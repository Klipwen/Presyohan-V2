PRESYOHAN MOBILE - ONBOARDING FLOW ARCHITECTURE AND IMPLEMENTATION PLAN

Prepared for: Presyohan Mobile
Scope: Onboarding Activity, Wizard Layout, Role-Based Navigation Paths, Local/Remote Record Validation, Post-Onboarding Action Triggers.

================================================================================
Objective
================================================================================
Design and document the implementation of a 3-step onboarding wizard inside the Presyohan Mobile application. The flow is designed to dynamically target first-time, record-free users upon verification or login, dividing them into distinct functional pathways (Store Owner/Staff vs. Customer) based on their business intent. It will pre-load setup behaviors (Create/Join store or Add Suki dialogs) on landing activities to ensure a frictionless transition from account creation to first productive action.

================================================================================
Deliverables
================================================================================

--------------------------------------------------------------------------------
Phase 1 – Current System Analysis
--------------------------------------------------------------------------------
- The codebase currently has no onboarding flow.
- Verified or authenticated users are routed directly from SplashActivity or LoginActivity into either StoreActivity (for store management) or CustomerHomeActivity (for customer searches/subscriptions) based on SessionManager preferences.
- This creates a feature gap where new users are dropped into empty dashboards with no initial setup guidance, requiring them to manually trigger the "Join/Create Store" bottom sheets.

--------------------------------------------------------------------------------
Phase 2 – Database Changes
--------------------------------------------------------------------------------
No database schema changes are required.
To verify if onboarding must run, the app will perform two remote checks using Supabase Postgrest client:
1. Check `store_members` table:
   Query: `supabaseClient.postgrest["store_members"].select { filter { eq("user_id", currentUserId) } }`
2. Check `suki_relationships` table:
   Query: `supabaseClient.postgrest["suki_relationships"].select { filter { eq("user_id", currentUserId) } }`

If both tables return empty lists for the authenticated user ID and the local SharedPreferences key `onboarding_completed` is false, the user is classified as a new user and onboarding is initiated.

Phase 4 – State Management
--------------------------------------------------------------------------------
1. In-Memory OnboardingState (housed in `OnboardingViewModel`):
   - `currentStep`: LiveData/StateFlow of Int (values: 1, 2, 3)
   - `userRole`: LiveData/StateFlow of UserRoleEnum (STORE, CUSTOMER, NONE)
   - `storeOption`: LiveData/StateFlow of StoreOptionEnum (CREATE, JOIN, NONE)
   - `sukiOption`: LiveData/StateFlow of SukiOptionEnum (YES, NO, NONE)

2. Persistent States (saved in SharedPreferences name `presyo_prefs`):
   - `onboarding_completed`: Boolean flag (true once "LAUNCH APP" is tapped).
   - `onboarding_action_pending`: String key (determines immediate main-screen behaviors):
     - `"create_store"`: Triggers CreateStoreDialog in StoreActivity.
     - `"join_store"`: Triggers JoinStoreDialog in StoreActivity.
     - `"add_suki"`: Triggers AddSukiDialog in CustomerHomeActivity.
     - `"select_presyohan"`: Routes directly to SelectPresyohanActivity.

Phase 6 – Navigation Flow
--------------------------------------------------------------------------------
```
[SplashActivity/LoginSuccess]
       |
       v (Verify Records in DB + Check completed flag)
       +---> [Records Found / Completed = True] -------> Route to Last Dashboard/Screen
       |
       +---> [No Records Found / Completed = False] ---> Route to OnboardingActivity
                                                                  |
                                                                  v
                                                        [Step 1: Choose Role]
                                                       /                     \
                                             (Store Selected)            (Customer Selected)
                                                     /                         \
                                                    v                           v
                                     [Step 2: Create or Join Store]     [Step 2: Do you have a Suki?]
                                                    \                           /
                                                     v                         v
                                             [Step 3: Store Success]   [Step 3: Customer Success]
                                                       \                       /
                                                        \                     /
                                                         v                   v
                                                      [User clicks LAUNCH APP]
                                                                 |
                                    +----------------------------+----------------------------+
                                    |                            |                            |
                             (Store Role)                 (Customer + Suki)             (Customer + No Suki)
                                    |                            |                            |
                                    v                            v                            v
                            Route to StoreActivity     Route to CustomerHomeActivity   Route to SelectPresyohanActivity
                            (Triggers Join/Create       (Triggers Add Suki Dialog)     (Direct Search/Browse)
                             dialog on launch)
```

Exit Modals & Cancelling:
- If a user triggers the back press during onboarding, they are shown the standardized confirmation dialog:
  - Title: "Exit Setup?"
  - Message: "Setup is not complete. Your account is saved, and you can finish setup later."
  - Positive Button: "Continue Setup" (Dismisses the modal, retains selection state)
  - Negative Button: "Exit App" (Calls `finishAffinity()` to exit the application).

Phase 8 – Risks and Edge Cases
--------------------------------------------------------------------------------
1. Edge Case: Network latency on login checks
   - Risk: Splashing is delayed or errors out because DB query takes too long.
   - Mitigation: Timeout splash checks to 3 seconds. If timeout or offline, check local SharedPreferences as fallback. If local metadata shows profile exists, skip onboarding.
2. Edge Case: App killed during onboarding
   - Risk: App process is killed by OS on Step 2.
   - Mitigation: On onboarding activity restart, read savedInstanceState or default to Step 1. Since completed flag is false, the user safely restarts onboarding from Step 1.
3. Edge Case: Account with pre-existing records logging into a clean device
   - Risk: Local completed flag is false on the clean device, which would launch onboarding.
   - Mitigation: The Splash check queries the live Supabase tables (`store_members`, `suki_relationships`). If records are found, it skips onboarding and updates the local completed flag to true.
